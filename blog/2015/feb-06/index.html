Creating a Responsive Website

All Articles

Next

Previous

When designing a website, you want to provide the best experience for everyone. By designing a responsive website, we assure it will respond correctly to the many different screens and devices. These kind of websites avoid any of the possible negative experiences the user can encounter when visiting a website on a small screen.

By today’s standards, responsive websites are almost a requirement. Mobile web is growing and with 80% (and counting) of internet users owning smartphones it’s crucial to a website’s success.

Responsive websites adapt to the browsers and the devices where they are being viewed. So when users go to a site on their smartphones, they will be able to view the same content and features that someone on a desktop sees. When a website is not responsive, it is much harder to use on different devices. Since mobile-friendly websites have become the norm, users are likely to quickly exit a site that doesn’t work easily on the device they’re using. The hassles of zooming and panning turns many users away when a site is not responsive.

When building a website, you have to make sure the website’s content and features are properly represented on desktop and mobile platforms. This means that copy is still legible when it goes to a smaller screen (because no one wants to struggle with text smaller than 9pt), and UI elements are still selectable for users with large fingers.

Most designers will assume that the screen size and the viewport of a browser or device the same, when they are actually different. Screen size determines the resolution of the device but the viewport determines how the website will be viewed. Understanding the viewport of a device is very useful to a designer, because knowing that affects how the designer wants the design to look on a specific device. Luckily there are many resources on the internet to figure out a device’s viewport size; my favorite is viewportsizes.com

This shows what the difference of an iPhone 6 Plus’ layout if it were based on the phone’s viewport and screen size.

There are always things to be considered for responsive websites. It’s my responsibility as an Interactive Designer to design the solutions and make sure that the website is a positive experience on every platform.
